# 钱包监控视图管理功能使用手册

## 一、功能概述

视图管理功能帮助您按照不同的筛选条件组织和查看钱包数据，类似于Excel的筛选器，让您可以快速切换不同的数据视图。

**核心价值**：

- 🎯 快速筛选：一键切换查看特定条件的钱包数据
- 💾 自动保存：视图配置自动保存，刷新页面后依然保留
- 🔄 灵活切换：支持创建最多50个自定义视图

---

## 二、视图标签栏

### 位置

表格上方，工具栏下方的Tab标签栏

### 默认视图

- **全部钱包**：显示所有钱包数据，无任何筛选条件

### 视图切换

点击不同的视图标签即可切换，数据会自动刷新并应用对应的筛选条件。

![视图标签栏示意]

```
┌─────────────┬──────────────┬───┐
│ 全部钱包 ✓  │ 高价值钱包   │ + │
└─────────────┴──────────────┴───┘
```

## 三、创建新视图

### 操作步骤

1. **点击"+"按钮**  
   位于视图标签栏最右侧

2. **填写视图信息**
   - **视图名称**（必填）：为视图起一个易于识别的名称，如"高价值钱包"
   - **筛选条件**（可选）：设置数据筛选规则

3. **配置筛选条件**

   支持的筛选字段：

   | 筛选字段     | 说明                 | 示例                    |
   | ------------ | -------------------- | ----------------------- |
   | 归属标签     | 多选，钱包归属类型   | 个人、团队、外部        |
   | 主链列表     | 多选，钱包所在区块链 | ETH、ARB、OP            |
   | 钱包总价值   | 数值范围，USD计价    | 最小值：20000           |
   | 地址活跃天数 | 数值范围             | 最小值：30，最大值：365 |
   | 活动标签     | 多选                 | 活跃、休眠、高频        |
   | 分类标签     | 多选                 | DeFi、NFT、GameFi       |
   | 状态标签     | 多选                 | 正常、监控中、已归档    |
   | 警报标记     | 多选                 | 高风险、异常交易        |

4. **其他设置**
   - **设为默认视图**：勾选后，每次进入页面自动激活此视图

5. **点击"确定"**  
   视图创建成功，自动切换到新视图

### 示例：创建"高价值钱包"视图

```
视图名称：高价值钱包
筛选条件：
  - 钱包总价值 ≥ 20000 USD
设为默认：否
```

创建后，此视图将只显示总价值大于等于20000美元的钱包。

## 四、管理视图

### 右键菜单操作

在视图标签上**右键点击**，弹出操作菜单：

| 操作            | 说明                               |
| --------------- | ---------------------------------- |
| 📝 **编辑**     | 修改视图名称或筛选条件             |
| 📋 **复制**     | 基于当前视图快速创建副本           |
| ⭐ **设为默认** | 设为默认视图，每次打开页面自动激活 |
| 🗑️ **删除**     | 删除视图（至少保留1个视图）        |

### 视图编辑

1. 右键点击视图标签 → 选择"编辑"
2. 修改视图名称或筛选条件
3. 点击"确定"保存

**注意**：如果编辑的是当前正在使用的视图，修改后数据会立即刷新。

### 视图复制

1. 右键点击视图标签 → 选择"复制"
2. 系统自动创建副本，名称为"原视图名（副本）"
3. 可以再次编辑副本进行微调

**使用场景**：想在现有视图基础上增加或修改少量筛选条件。

### 删除视图

1. 右键点击视图标签 → 选择"删除"
2. 确认删除
3. 如果删除的是当前视图，系统自动切换到第一个视图

**限制**：至少保留1个视图，最后一个视图无法删除。

## 五、使用技巧

### 1. 快速筛选特定数据

**场景**：快速查看符合特定条件的钱包

**操作**：

- 创建视图时设置精确的筛选条件
- 例如："监控中的高价值钱包" = 总价值≥10000 + 状态=监控中

### 2. 数值范围筛选

**钱包总价值筛选示例**：

| 需求                 | 配置                          |
| -------------------- | ----------------------------- |
| 大额钱包（≥100万）   | 最小值：1000000，最大值：留空 |
| 中等价值（1万-10万） | 最小值：10000，最大值：100000 |
| 小额钱包（<1000）    | 最小值：留空，最大值：1000    |

### 3. 多条件组合筛选

**场景**：查找个人持有的、在以太坊上的、价值超过5万的活跃钱包

**配置**：

```
归属标签：个人
主链列表：ETH
钱包总价值：≥50000
活动标签：活跃
```

### 4. 视图命名建议

好的命名让视图一目了然：

✅ **推荐**：

- "高价值钱包（≥2万）"
- "个人DeFi钱包"
- "需关注-异常交易"

❌ **不推荐**：

- "视图1"
- "test"
- "临时"

## 六、常见问题

### Q1：视图最多可以创建多少个？

**A**：最多50个视图。建议定期清理不常用的视图。

### Q2：刷新页面后视图会丢失吗？

**A**：不会。视图配置自动保存在浏览器本地存储中，刷新页面后依然保留。

### Q3：切换视图后分页会重置吗？

**A**：是的。每次切换视图都会重置到第一页，并重新加载数据。

### Q4：可以导出/导入视图配置吗？

**A**：暂不支持。视图配置仅保存在当前浏览器中。

### Q5：删除视图后能恢复吗？

**A**：不能。删除操作不可逆，请谨慎操作。

### Q6：视图筛选和搜索栏筛选有什么区别？

**A**：

- **视图筛选**：保存后可重复使用，适合常用的筛选条件
- **搜索栏筛选**：临时筛选，刷新后失效，适合一次性查询

### Q7：为什么视图显示的数据条数和实际不符？

**A**：可能的原因：

1. 后端SQL COUNT查询与实际筛选结果略有差异（技术限制）
2. 数据在查询过程中被修改或删除
3. 分页偏移量计算问题

建议以实际显示的数据为准。

## 七、注意事项

### 浏览器兼容性

- ✅ Chrome 90+
- ✅ Edge 90+
- ✅ Firefox 88+
- ✅ Safari 14+

### 存储限制

- 视图配置存储在浏览器LocalStorage中（最大5MB）
- 清除浏览器数据会导致视图配置丢失
- 更换浏览器或电脑需要重新创建视图

### 数据安全

- 视图配置仅保存在本地浏览器中，不会上传到服务器
- 清除浏览器缓存会删除所有视图配置

## 八、快捷操作参考

## 九、示例场景

### 场景1：监控高风险钱包

**需求**：快速查看所有标记为"高风险"或"异常交易"的钱包

**操作**：

1. 创建视图："高风险监控"
2. 设置筛选：警报标记 = 高风险、异常交易
3. 保存并切换到该视图

### 场景2：统计个人DeFi资产

**需求**：查看个人持有的所有DeFi相关钱包

**操作**：

1. 创建视图："个人DeFi"
2. 设置筛选：
   - 归属标签 = 个人
   - 分类标签 = DeFi
3. 保存并切换到该视图

### 场景3：定期检查大额钱包

**需求**：每天检查价值超过10万美元的钱包活动

**操作**：

1. 创建视图："大额钱包监控"
2. 设置筛选：钱包总价值 ≥ 100000
3. 设为默认视图（每次打开页面自动显示）
